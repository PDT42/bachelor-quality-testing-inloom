TestDataSets on Record:

"These are the TestDataSets registered in the application database. Each TestDataSet is based on one student solution. A student solution is unique, just like each TestDataSet is unique. For each student solution there can exist multiple manual and automatic evaluations, which is why each TestDataSet holds a list of both: ManEvals and AutoEvals. When "running" the test case, each of the ManEvals is considered, while only the AutoEval, that was recorded last is processed."

Register new Exercise

Exercises on Record:

"These are the exercises recorded in the application database. Each of the student solutions (rather obviously) aims to solve a given task, for which there is at least one optimal solution. When grading the student solution, the evaluator compares it to an expert solution and awards points, if the required features are part of the student solution. For one task, there can exist multiple expert/optimal solutions, since modelling is not an exact science."

Registered Evaluators:

"Every evaluator has his/her/their own style. Some grade more harshly, some are more lenient. Some have good, some, bad days. Thus, the manual evaluations of two different evaluators are not equal and should not be treated as such. These are the evaluators registered in the system."
